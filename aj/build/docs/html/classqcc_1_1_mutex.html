QStatus qcc::Mutex::Lock ( const char *  file,
uint32_t  line 
)

Acquires a lock on the mutex.

If another thread is holding the lock, then this function will block until the other thread has released its lock.

NOTE: Best practice is to call Mutex::Lock(MUTEX_CONTEXT)

See also
MUTEX_CONTEXT
Parameters
filethe name of the file this Mutex lock was called from
linethe line number of the file this Mutex lock was called from
Returns
  • ER_OK if the lock was acquired.
  • ER_OS_ERROR if the underlying OS reports an error.

bool qcc::Mutex::TryLock ( )

Attempt to acquire a lock on a mutex.

If another thread is holding the lock this function return false otherwise the lock is acquired and the function returns true.

Returns
True if the lock was acquired.

uint32_t  line

Releases a lock on the mutex.

This will only release a lock for the current thread if that thread was the one that aquired the lock in the first place.
