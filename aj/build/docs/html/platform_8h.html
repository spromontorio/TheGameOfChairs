#define QCC_UNUSED (   x)    (void)(x)

Macro used to avoid unused variable warning in release code.

This Macro is only used when a variable is only used in the debug build variant. When building in release mode the compiler will give an unused variable warning.

Example usage:

1 String errMsg
2 const char* errName = reply->GetErrorName(&errMsg);
3 QCC_LogError(status, ("TimedPing returned ERROR_MESSAGE (error=%s, \"%s\")", errName, errMsg.c_str()));
4 QCC_UNUSED(errName); // avoid unused variable warning in release build

The QCC_LogError is an empty macro for release build variants but in debug build variants it is not empty so it uses the errName variable.
